# DASIDA Superpowers

이 스킬은 다음 상황에서 사용합니다:
- 새 기능 아이디어를 스펙으로 만들 때 → `superpowers:brainstorming`
- 구현 계획 파일을 단계별로 실행할 때 → `superpowers:executing-plans`
- 서브에이전트를 활용해 병렬로 구현할 때 → `superpowers:subagent-driven-development`

## 파일 구조

| 경로 | 역할 |
|------|------|
| `docs/superpowers/specs/YYYY-MM-DD-<slug>-design.md` | 브레인스토밍 결과 — 스펙/디자인 문서 |
| `docs/superpowers/plans/YYYY-MM-DD-<slug>.md` | 구현 계획 — 체크박스 태스크 목록 |

## superpowers:brainstorming

**트리거**: 새 기능 아이디어, "이거 만들면 어때요", "스펙 써줘", 제품 개선 제안

### 작업 순서

1. 아이디어를 듣고 관련 기존 파일 확인 (`docs/superpowers/specs/`, `docs/superpowers/plans/`, 관련 feature 코드)
2. 스펙 문서 작성 → `docs/superpowers/specs/YYYY-MM-DD-<slug>-design.md`
3. 구현 계획 작성 → `docs/superpowers/plans/YYYY-MM-DD-<slug>.md`
4. Notion DASIDA 개발 기록에 초안 페이지 생성 (상태: 기획중) — PostToolUse 훅이 리마인더를 출력하면 즉시 실행

### 스펙 파일 형식

```markdown
# Spec: <기능명>

Generated by /superpowers:brainstorming on YYYY-MM-DD
Branch: <현재 브랜치>
Repo: dasida-app
Status: DRAFT

## 한 줄 요약
<한 문장으로>

---

## 배경
<왜 필요한가>

---

## 스펙
<상세 설계>

---

## 변경 파일 목록
| 파일 | 변경 내용 |
|------|----------|

---

## Success Criteria
- <검증 항목들>
```

### 구현 계획 파일 형식

```markdown
# <기능명> Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** <목표>
**Architecture:** <구조 설명>
**Tech Stack:** <기술 스택>

---

## Task 1: <태스크명>

**Files:**
- Create/Modify: `<파일 경로>`

- [ ] **Step 1: <작업 내용>**
- [ ] **Step 2: TypeScript 타입 확인**
```bash
npx tsc --noEmit
```
- [ ] **Step 3: 커밋**
```bash
git add <files>
git commit -m "<message>"
```
```

### 규칙

- 파일명: `YYYY-MM-DD-<slug>.md` (오늘 날짜 기준)
- slug는 영어 소문자 kebab-case
- 스펙과 플랜은 항상 쌍으로 작성
- Status는 DRAFT로 시작, 검토 후 APPROVED로 변경

## superpowers:executing-plans

**트리거**: "이 플랜 실행해줘", "구현해줘", 특정 plan 파일 지정

### 작업 순서

1. `docs/superpowers/plans/` 에서 대상 plan 파일 읽기
2. 관련 스펙 파일 (`docs/superpowers/specs/`) 확인
3. 체크박스(`- [ ]`) 순서대로 Task 실행
4. 각 Step 완료 후 plan 파일의 체크박스를 `- [x]`로 업데이트
5. TypeScript 확인 (`npx tsc --noEmit`) 후 커밋

### 규칙

- Task는 순서대로 실행 (병렬 금지, 의존성 있음)
- 각 Step 완료 즉시 체크박스를 업데이트
- TypeScript 에러 발생 시 다음 Step으로 넘어가지 않음
- 커밋은 plan 파일에 명시된 메시지 형식을 사용

---

## superpowers:subagent-driven-development

**트리거**: "서브에이전트로 구현해줘", 복잡한 multi-task plan

### 작업 순서

1. Plan 파일을 읽고 독립적으로 실행 가능한 Task를 파악
2. 의존성 없는 Task는 Agent 툴로 병렬 실행
3. 의존성 있는 Task는 순서대로 직렬 실행
4. 각 서브에이전트 결과를 확인하고 통합
5. 최종 `npx tsc --noEmit` 확인 및 커밋

### 규칙

- 각 서브에이전트는 단일 Task만 담당
- 서브에이전트에게 관련 스펙 파일 + plan Task 내용을 함께 전달
- 서브에이전트 결과는 반드시 검증 후 다음 단계로 진행
