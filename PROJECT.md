# DASIDA 제품 전략

> 10문제 체험 -> 매일 약점 학습 -> 실전 모의고사
>
> 3분만에 내 약점을 찾고, 매일 짧게 보완하고, 실제 모의고사에서 다시 확인한다.

---

## 1. 제품 한 줄 정의

DASIDA는 `문제를 많이 푸는 앱`이 아니라, `내 약점을 빠르게 진단하고 반복 학습으로 실전에 연결하는 앱`이다.

핵심 경험은 아래 3단계로 정의한다.

1. 10문제로 빠르게 약점을 찾는다.
2. 그 약점을 매일 짧게 다시 푼다.
3. 실제 모의고사에서 다시 확인한다.

즉 제품의 중심은 `진단`이 아니라 `진단 이후의 반복 학습`이다.

---

## 2. 왜 10문제인가

10문제는 콘텐츠의 완결판이 아니라 활성화 장치다.

10문제가 해야 하는 역할은 아래와 같다.

- 설치 직후 부담 없이 시작하게 한다.
- 사용자의 약점을 빠르게 개인화한다.
- 개인화된 결과를 근거로 다음 행동을 제안한다.
- 재방문 이유를 만든다.

정리하면:

- `10문제 = activation funnel`
- `약점 연습 = retention engine`
- `모의고사 = credibility + expansion engine`

따라서 10문제는 메인 제품이 아니라 `첫 진입 경험`으로 설계해야 한다.

---

## 3. 현재 앱 상태

현재 저장소 기준 사실은 다음과 같다.

- 메인 퀴즈 탭은 `10문제 약점 진단`을 바로 시작하는 구조다.
- 결과 화면은 `상위 약점 3개`와 `약점 연습` 또는 `심화 문제`로만 이어진다.
- 연습 화면은 `practiceQueue` 기반 약점 문제 재풀이 구조다.
- `history` 탭은 아직 비어 있다.
- 현재 데이터는 10문제 본문, 약점별 연습문제, 1개의 challenge 문제만 있다.
- 최근 3년 모의고사 데이터 구조와 시도 기록 구조는 아직 없다.

즉 지금 앱은 `체험`은 있지만, `정착`과 `실전 확장`의 껍데기가 아직 없다.

### 지금 구조의 한계

- 10문제를 다 풀고 나면 다시 들어올 이유가 약하다.
- 앱의 핵심 가치가 `짧은 체험`으로만 느껴진다.
- 사용자가 `이 앱이 실전에도 쓸 만하다`고 느낄 근거가 부족하다.
- 모의고사나 장기 학습으로 이어지는 동선이 아직 약하다.

---

## 4. 새 제품 구조

앱의 중심을 `10문제 퀴즈 앱`에서 `약점 학습 허브`로 바꾼다.

### 핵심 구조

`10문제 체험 -> 결과 -> 오늘의 약점 학습 -> 실전 모의고사`

### 새 사용자 여정

1. 숏폼에서 앱 설치
2. 앱 첫 화면에서 `3분 체험 10문제`를 가장 크게 제안
3. 체험 후 즉시 `내 약점 1~3개`를 보여줌
4. 바로 `오늘의 약점 학습`으로 이어짐
5. 동시에 `실전 모의고사 다시 풀어보기` CTA 노출
6. 이후 재방문 시 첫 화면은 더 이상 10문제가 아니라 `오늘 학습 / 실전 다시 풀기 / 최근 진단 결과`가 됨

즉:

- 첫 실행은 `체험 중심`
- 두 번째 실행부터는 `학습 중심`

## 5. 출시 기본값

첫 공개 버전은 아래 범위로 고정한다.

### 반드시 포함

- 10문제 체험
- 오답 약점 분석
- 약점별 매일 학습
- 알림 opt-in
- 대표 모의고사 1개 묶음

### 포함하지 않음

- 최근 3년 전체 고1/고2/고3 아카이브
- 완전한 리포트/기록 분석 화면
- 웹 중심 체험 플로우

### 이 범위를 택하는 이유

- 10문제만으로는 앱의 쓰임새가 약하다.
- 최근 3년 전체를 처음부터 넣으면 콘텐츠와 상태 설계 범위가 너무 커진다.
- 첫 공개에서는 `실전까지 이어지는 느낌`만 증명하면 충분하다.

### 2단계 확장

- 고1 최근 3년 모의고사
- 고2 최근 3년 모의고사
- 고3 최근 3년 모의고사
- 회차별 아카이브
- 시험별 오답 복기 리포트
- 누적 히스토리와 성장 리포트

## 6. 정보 구조와 라우트 방향

현재 탭 구조는 유지하되, `문제 풀기` 탭의 성격을 바꾼다.

### 탭 구조

- `문제 풀기`
  - 학습 허브
- `내 기록`
  - 차후 리포트 확장을 위한 자리
- `설정`
  - 알림 시간, 학년, 목표, 데이터 관리

### 문제 풀기 탭의 섹션 순서

1. 오늘의 약점 학습
2. 3분 체험 10문제
3. 실전 모의고사
4. 최근 학습 요약

즉 이 탭은 더 이상 `바로 10문제 시작`만 의미하지 않는다.

### 라우트 재구성 방향

- `/quiz/index`
  - 학습 허브
- `/quiz/diagnostic`
  - 현재 10문제 체험 플로우
- `/quiz/result`
  - 진단 결과 요약
- `/quiz/practice`
  - 약점 학습
- `/quiz/exams`
  - 모의고사 목록
- `/quiz/exams/[examId]`
  - 특정 모의고사 풀이
- `/quiz/exam-result`
  - 모의고사 결과와 오답 복기
- `/quiz/feedback`
  - 최종 피드백

## 7. 필요한 도메인 타입

아래 타입은 앞으로 모의고사와 재방문 흐름을 설계할 때 기준으로 사용한다.

### 시험 메타데이터

```ts
type ExamGrade = 'g1' | 'g2' | 'g3';

type ExamMeta = {
  id: string;
  grade: ExamGrade;
  year: number;
  month: 3 | 6 | 9 | 11;
  title: string;
  questionCount: number;
  durationMinutes?: number;
  isFeatured: boolean;
};
```

### 모의고사 문항

```ts
type ExamQuestion = {
  id: string;
  examId: string;
  number: number;
  question: string;
  choices: string[];
  answerIndex: number;
  topic: string;
  diagnosisMethods: SolveMethodId[];
  candidateWeaknesses: WeaknessId[];
};
```

### 모의고사 세션 상태

```ts
type ExamAttempt = {
  examId: string;
  grade: ExamGrade;
  startedAt: string;
  completedAt?: string;
  answers: Array<{
    questionId: string;
    selectedIndex: number;
    isCorrect: boolean;
    weaknessId?: WeaknessId;
  }>;
  score: number;
  topWeaknesses: WeaknessId[];
};
```

### 일일 학습 태스크

```ts
type DailyReviewTask = {
  id: string;
  weaknessId: WeaknessId;
  source: 'diagnostic' | 'exam';
  scheduledFor: string;
  stage: 'day1' | 'day3' | 'day7' | 'day14';
  completed: boolean;
};
```

### 홈 화면 추천 상태

```ts
type HomeLearningState = {
  hasCompletedDiagnostic: boolean;
  activeWeaknessIds: WeaknessId[];
  nextReviewTask?: DailyReviewTask;
  featuredExamId?: string;
};
```

## 8. 저장 전략

현재 진단 데이터는 세션 상태 중심이다. 하지만 새 전략으로 가면 아래 데이터는 세션을 넘어 저장해야 한다.

### 저장해야 하는 것

- 최근 10문제 진단 결과
- 확정된 상위 약점 1~3개
- 일일 학습 태스크
- 모의고사 시도 기록
- 마지막 학습 날짜
- 알림 opt-in 여부와 시간

### 1단계 저장소

- 로컬 저장: AsyncStorage 또는 Expo 권장 영속 저장

이유:

- 초반에는 서버 동기화보다 재방문 경험이 더 중요하다.
- 계정 없이도 동작하는 것이 MVP 확장에 유리하다.

### 2단계 저장소

- Firebase/Firestore 동기화

적용 시점:

- 계정 기반 히스토리
- 장기 리포트
- 디바이스 간 동기화

## 9. 콘텐츠 전략

### 10문제 콘텐츠의 역할

- 빠른 진단
- 첫 성공 경험
- 개인화의 시작점

유지 원칙:

- 10문제는 계속 유지한다.
- 학년 통합 공통 체험판으로 운영한다.
- 너무 길어지지 않도록 유지한다.

### 모의고사 콘텐츠의 역할

- 실전성 증명
- 반복 학습의 목적지 제공

첫 공개 버전 기본값:

- 대표 모의고사 1개 묶음만 제공

추천안:

- 고2 최근 1회 또는 대표성 높은 1개 세트

이유:

- 난도와 범용성이 가장 적절하다.
- 너무 많은 시험을 한 번에 보여주지 않아도 실전성은 증명할 수 있다.

### 2단계 확장

- 고1, 고2, 고3 각각 확장
- 최근 3년 회차별 정리
- 시험별 오답 복기 연결

## 10. 알림 전략

에빙하우스 망각곡선을 참고하되, 처음부터 복잡하게 가지 않는다.

### 1단계 알림 규칙

- 진단 또는 모의고사 완료 후
- 상위 약점 1개 기준으로
- Day 1, Day 3, Day 7 알림만 생성

### 알림 문구 예시

- `어제 찾은 약점을 3분만 다시 볼까요?`
- `잊기 전에 한 문제만 다시 풀어보세요`
- `실전에서 흔들리는 부분을 오늘 다시 잡아볼 시간이에요`

### 2단계 확장

- 약점별 적응형 빈도
- 시험 일정 기반 리마인더
- 시험 직전 복습 패키지

## 11. 구현 단계

### Phase 1 — 제품 구조 재정의

목표:

- 앱이 `10문제 앱`이 아니라 `약점 학습 앱`처럼 보이게 만든다.

포함:

- `quiz/index`를 학습 허브로 전환
- `quiz/diagnostic`로 현재 10문제 플로우 이동
- 홈에 `오늘의 약점 학습`, `3분 체험`, `실전 모의고사` 카드 배치
- 진단 완료 후 결과에서 `실전 모의고사 다시 풀기` CTA 추가

### Phase 2 — 일일 학습과 알림

목표:

- 재방문 이유를 만든다.

포함:

- 진단 결과 영속 저장
- 일일 review task 생성
- 알림 opt-in 플로우
- 홈에 `오늘 할 학습 1개` 노출

### Phase 3 — 대표 모의고사 1개 묶음

목표:

- `실전에도 쓸 만하다`는 감각을 증명한다.

포함:

- `ExamMeta`, `ExamQuestion`, `ExamAttempt` 도입
- 모의고사 목록 1개
- 시험 풀이 화면
- 시험 결과에서 오답 약점 연결

### Phase 4 — 최근 3년 아카이브

목표:

- 고1/고2/고3 확장

포함:

- 학년/연도/월별 목록
- 아카이브 탐색
- 시험별 재도전
- 누적 기록과 리포트

## 12. 사용자가 체감하는 순서

### 설치 첫날

1. 앱 설치
2. 10문제 체험
3. 내 약점 보기
4. 오늘의 약점 학습 1개
5. 실전 모의고사 다시 풀기 제안

### 재방문 2~7일

1. 오늘의 약점 학습
2. 전날 놓친 약점 다시 보기
3. 대표 모의고사 다시 풀기
4. 필요하면 10문제 재진단

이 순서를 지켜야 `체험 -> 반복 -> 실전`이 자연스럽게 이어진다.

## 13. 사용자가 모의고사를 다시 풀고 싶게 만드는 장치

모의고사는 별도 콘텐츠가 아니라 `진단의 다음 행동`으로 보여줘야 한다.

### 결과 화면 직후 메시지

- 제목: `지금 가장 먼저 잡아야 할 약점`
- 본문: `이 약점은 실전 모의고사에서도 반복되기 쉬워요. 오늘은 짧게 보완하고, 그다음 실제 문제에서 다시 확인해보세요.`
- 1차 CTA: `오늘의 약점 학습 시작`
- 2차 CTA: `실전 모의고사 다시 풀기`

### 모의고사 CTA 문구 예시

- `방금 찾은 약점이 실전에서 어떻게 무너지는지 확인해보세요`
- `최근 모의고사에서 같은 유형을 다시 풀어볼 수 있어요`
- `진단만 끝내지 말고 실전에서 다시 써먹어보세요`

- D1 재방문율

### Expansion

- 10문제 완료 후 모의고사 진입률
- 모의고사 시작률
- 모의고사 완료율
- 모의고사 후 재학습 진입률

## 15. 명시적 가정과 기본값

- 핵심 제품 가치는 `약점 진단`보다 `약점 기반 반복 학습`이다.
- 10문제는 메인 제품이 아니라 activation 도구로 둔다.
- 최근 3년 모의고사는 반드시 가야 하지만, 첫 공개 버전에서 전체를 한 번에 넣지 않는다.
- 첫 공개 버전 기본값은 `10문제 체험 + 매일 약점 학습 + 알림 + 대표 모의고사 1개 묶음`이다.
- 유입은 `숏폼 -> 앱 바로 설치`를 기본값으로 둔다.
- 웹은 현재 기준으로 주력 학습 채널이 아니라 보조 랜딩/설명 채널로 둔다.
- 리포트와 히스토리 화면은 중요하지만, 이번 전략에서는 후순위다.
- 홈 화면의 첫 목적은 `무엇을 할지 보여주는 것`이 아니라 `오늘 왜 다시 들어와야 하는지 보여주는 것`이다.
