## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 앱 이름 | DASIDA |
| 타겟 | 고3 수험생 (수능 준비 중인 현역) |
| 핵심 가치 | 단순 문제 풀이가 아니라 '내 약점을 알고 그걸 바로 훈련한다'는 경험 제공 |
| 개발 범위 (MVP) | 고1 공통수학 데이터 기반 (추후 고3 공통과목으로 교체) |
| 기술 스택 | React Native + Expo + Firebase |
| AI 판정 | OpenAI API (response_format 활용) |
| 배포 방식 | 앱스토어 + 구글 플레이 (OTA 업데이트 활용) |
| 팀 | 기윤 (기획/개발/마케팅), 가민 (디자인) |

## 3. 사용자 흐름 (User Flow)

```
[앱 실행]
   ↓
[문제 출제] → 10문제 중 1개 (고1 공통수학)
   ↓
[답 입력] → 객관식 선택 또는 직접 텍스트 입력
   ↓
[AI 판정] → correct / partial / wrong 3분기
   ↓
[약점 태그 표시] → 예: '계산 실수 반복', '공식 이해 부족'
   ↓
[연습문제 1개 제공] → 해당 약점 기반 훈련 문제
   ↓
[피드백 입력] → '어땠나요? 한 줄만 남겨주세요'
   ↓
[Firebase 저장] → 기윤에게 알림
```

## 4. 약점 태그 목록 (9개)

기존 index.html `diagnosisMap`에 이미 정의되어 있음.

| 약점 태그 | 설명 | 연습문제 방향 |
|-----------|------|---------------|
| 공식 이해 부족 | 완전제곱식 변환 원리 미숙 | 완전제곱식 변환 기본 1문제 |
| 계산 실수 반복 | 개념은 알지만 계산 과정 실수 | 부호 포함 계산 정확도 훈련 |
| 최솟값 읽기 혼동 | (x-a)²+b에서 b vs a 혼동 | 꼭짓점 좌표 읽기 집중 문제 |
| 공식 암기 부족 | 꼭짓점 공식 -b/2a 불안정 | 공식 적용 반복 문제 |
| 계수 구분 혼동 | a, b, c 부호 포함 읽기 실수 | 계수 추출 집중 문제 |
| 미분 계산 부족 | xⁿ → nxⁿ⁻¹ 규칙 불안정 | 미분 기본 계산 문제 |
| 풀이 순서 혼동 | f'(x)=0 → 대입 순서 누락 | 2단계 풀이 순서 확인 문제 |
| 최댓값/최솟값 판단 혼동 | a 부호로 판별 못함 | a 부호 → 볼록 방향 판별 문제 |
| 기초 개념 학습 필요 | 이차함수 최솟값 개념 미숙 | 개념 설명 + 기초 문제 |

## 5. AI 응답 구조

### 판정 방식

| 판정값 | 의미 | 다음 화면 |
|--------|------|-----------|
| correct | 정답 | 정답 결과 + 풀이 스타일 분석 |
| partial | 방향은 맞지만 미완성 | 힌트 제공 + 재시도 유도 |
| wrong | 오답 | 약점 태그 표시 + 연습문제 제공 |

### AI 호출 응답 형식 (OpenAI API)

```json
{
  "nextStep": "correct | partial | wrong",
  "message": "피드백 텍스트"
}
```

## 6. 데이터 구조

### practiceMap (약점별 연습문제, 하드코딩)

```js
const practiceMap = {
  "계산 실수 반복": {
    question: "문제 텍스트",
    choices: ["①", "②", "③", "④", "⑤"],
    answer: "②",
    hint: "힌트 텍스트"
  },
  // 나머지 8개 태그도 동일 구조
};
```

**방법 A (MVP 채택):** 하드코딩 — 태그별 문제 미리 제작
**방법 B (나중):** AI 실시간 생성 — 비용 발생, MVP에서는 불필요

## 9. 예외 처리 (MVP 범위)

### 반드시 처리 (앱이 터지는 경우)

```js
// AI 호출
try {
  const result = await callAI(userInput);
  handleResult(result);
} catch (error) {
  showMessage('잠시 후 다시 시도해주세요');
}

// 피드백 제출
if (feedbackText.trim() === '') {
  showMessage('한 줄만 적어주세요');
  return;
}
```

### MVP에서 처리하지 않아도 되는 것

| 상황 | 이유 |
|------|------|
| 이상한 텍스트 입력 | AI가 자동으로 wrong 분류 |
| 중복 제출 | Firebase에 그냥 쌓으면 됨 |
| 앱 강제 종료 후 복구 | MVP 범위 외 |
| 오프라인 저장 | MVP 범위 외 |
