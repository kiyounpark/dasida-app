## 한 줄 결론

진짜 손볼 것 **1건**, 살짝 좁지만 MVP에서 둬도 되는 것 14건, 나머지 116건 문제 없음.

## 검사자 2명이 엇갈린 지점 (이게 교차검증의 핵심)

1차(매핑검사자)는 보충 흐름의 **맨 앞 5장 제목만** 보고 32 weak / 2 mismatch 로 깠다.
2차(수포자학생)가 실제 보충 파일을 열어보니, 대부분 학생이 누른 경로 뒤쪽 카드에
호소한 세부 내용이 들어있어 안 황당했다. 코드 직접 확인 결과 2차가 옳았다.

→ 1명이었으면 거짓 경보로 헛수고/개악할 뻔. 2명 교차가 막았다.

## ⚠️ 사람 결정 필요 — 1건

### 1. `linear_eq/__fallback__` — 함수에서 막혔는데 이차부등식 보충

- 풀이법 이름: **"부등식·함수"** (두 주제를 한 풀이법으로 묶음)
- 폴백 약점: `g2_inequality_range` (이차부등식 범위 오류)
- 문제: 이 풀이법을 고른 학생이 확인문제를 연속으로 틀리면 폴백으로 떨어지는데,
  학생이 **함수** 쪽에서 막혔어도 **이차부등식** 보충을 받게 됨. 딴소리로 느껴짐.
- 선택지(사람 판단):
  - (A) 그대로 둔다 — 폴백은 드물게만 도달, MVP 영향 작음
  - (B) 폴백 약점을 더 중립적인 것으로 교체
  - (C) "부등식"과 "함수"를 별도 풀이법으로 분리 (구조 변경, 범위 큼)
- 권장: MVP 단계면 (A) 또는 (B). (C)는 별도 작업.

약점에 여러 하위기능 호소를 몰아넣음. 출시 후 로그 보고 보강 여부 판단 권장.
