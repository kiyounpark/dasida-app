## Problem Statement

수험생들이 아이패드로 공부를 많이 한다. 현재 DASIDA는 supportsTablet: true이지만
오답 진단 세션 화면이 아이패드에서도 아이폰 FlatList 스와이프 방식 그대로 렌더링된다.
아이패드의 넓은 화면을 활용하지 못하고 있다.

## Target User

수능 수학 수험생 (고3). iPad를 GoodNotes, 유튜브 강의와 함께 사용.
DASIDA를 Split View로 GoodNotes 옆에서 사용할 가능성 있음.

## Constraints

- 기존 Expo/React Native 코드베이스 유지
- 사이드바(NavigationRail) 제외 — Bottom Tabs 유지
- Apple Pencil 지원: 이번 범위 밖
- Split View 안전성: 744pt 미만이면 자동으로 아이폰 레이아웃으로 fallback

## 핵심 설계

### useIsTablet 훅 (기존 코드 그대로 사용)

```tsx
// hooks/use-is-tablet.ts — 이미 존재, 수정 불필요
import { useWindowDimensions } from 'react-native';
export function useIsTablet(): boolean {
  const { width } = useWindowDimensions();
  return width >= 744; // iPad mini portrait(744pt) 포함
}
```

### ExamDiagnosisSessionScreen — 아이패드 레이아웃

```
[아이폰 — 변경 없음]
┌──────────────┐
│ DiagnosisDarkHeader (진행 점 포함) │
├──────────────┤
│  FlatList    │  ← 기존 수평 스와이프 그대로
│  (스와이프)   │
└──────────────┘

[아이패드 — 신규]
┌─────────────────────────────────────────────┐
│ DiagnosisDarkHeader (문제 번호, 뒤로 가기만) │
├──────────────────────┬──────────────────────┤
│                      │                      │
│   ExamDiagnosisPage  │   ExamDiagnosisPage  │
│   (문제 이미지 패널)  │   (답 입력 패널)     │
│   flex: 0.6          │   flex: 0.4          │
│                      │                      │
│  ● ○ ○ ○ (진행 점)  │   [다음 문제 →]      │
└──────────────────────┴──────────────────────┘
```

**동작:**
- `isTablet = true`: FlatList 제거, 현재 activeProblemIndex 문제만 렌더링
- 진행 점(dots): 왼쪽 패널 하단 — 탭하면 해당 문제로 이동 (인터랙티브 유지)
- `isTablet = false`: 기존 FlatList 스와이프 완전히 동일

### Split View 자동 대응

iPad Split View 50% 너비 ≈ 560pt → 744pt 미만 → isTablet = false → 아이폰 레이아웃 자동 fallback. 추가 코드 불필요.

### iPad mini 세로 방향

width ≈ 744pt → isTablet = true (포함). 가로 회전 시에도 동일하게 2-컬럼.
