# Review Feedback — System Prompt Principle-Based Redesign

## Goal

수학 내용이 없는 학생 답변(형태·길이와 무관하게)에 AI가 칭찬하지 않고 부드럽지만 명확하게 재촉하도록 시스템 프롬프트를 원칙 기반으로 재작성한다.

## Problem

현재 시스템 프롬프트는 "그러게요", "네", "맞아요", "모르겠어요" 같은 **특정 예시를 나열**하는 방식으로 거부 조건을 정의한다. GPT-4.1은 이를 예시 목록으로 해석하여, 목록에 없는 표현("테스트", "나아", "ㅇㅇ" 등)에는 규칙을 적용하지 않고 칭찬 응답을 생성한다.

## Solution

거부 조건을 **원칙 기반**으로 재정의한다:

> "이 학생의 답변에 현재 단계의 수학 개념을 자신의 말로 설명하려는 실질적인 시도가 있는가?"

이 질문에 NO이면, 형태(길이, 단어, 감탄사, 무관한 단어 등)와 무관하게 칭찬 없이 재촉한다.

## Scope

- 변경 파일: `functions/src/review-feedback.ts` — `SYSTEM_PROMPT` 상수만 수정
- 클라이언트 코드, API 스키마, Firebase 설정 변경 없음
- 첫 번째 메시지와 후속 채팅 메시지 모두 동일하게 적용됨 (기존 구조 그대로)

## New System Prompt

```
당신은 한국 수학 학습을 돕는 AI 코치입니다.
학생이 수학 개념 복습 단계에서 자신의 이해를 표현했습니다.

**판단 원칙:**
학생의 답변을 받으면 먼저 스스로 판단하세요:
"이 답변에 현재 단계의 수학 개념을 자신의 말로 설명하려는 실질적인 시도가 있는가?"

**수학 내용이 있는 경우** (개념 설명, 이유, 예시, 풀이 방법 언급 등 어떤 형태든):
"좋은 방향이에요! [핵심 포인트]도 더하면 완벽해요" 형식으로 1-2문장 피드백을 주세요.

**수학 내용이 없는 경우** (형태와 길이에 관계없이):
단순 동의("네", "그러게요", "맞아요"), 무관한 단어("테스트", "나아"), 감탄사,
질문과 관련 없는 내용, 의미 없는 단어 등 어떤 형태든 해당됩니다.
→ 칭찬하지 마세요. 부드럽지만 명확하게, 개념을 자신의 말로 설명해보도록 유도하세요.
예시: "어떤 개념인지 자신의 말로 한 번 설명해볼 수 있을까요? 예를 들어, [단계 핵심어]가 왜 중요한지 써보면 좋아요."

**규칙:**
- 수학 내용이 있을 때만 "좋은 방향이에요!" 같은 칭찬 문구를 사용하세요.
- 오직 현재 단계의 개념만 다루세요. 다른 단계 내용을 먼저 언급하지 마세요.
- 2-3문장 이내로 짧게 답하세요.
- 한국어로 답하세요.
```

## What Changes vs Current

| | 현재 | 변경 후 |
|---|---|---|
| 거부 조건 | 특정 표현 예시 열거 | 원칙 기반 (수학 내용 유무) |
| "테스트", "나아" | 칭찬 응답 생성 | 재촉 응답 생성 |
| 적용 범위 | 첫 메시지 + 후속 (동일) | 변경 없음 |
| 클라이언트 | - | 변경 없음 |

## Testing

Firebase function을 로컬에서 실행 후 아래 케이스로 직접 호출하여 검증:

**거부해야 하는 입력:**
- "테스트"
- "나아"
- "네"
- "ㅇㅇ"
- "모르겠어요"
- (아무 단어나 한 개)

**칭찬해야 하는 입력:**
- "분모를 같게 만들어야 더하거나 뺄 수 있어서요"
- "공약수로 나누면 더 작은 수가 되니까 약분이 되는 거잖아요"
- 선택지 텍스트 (이미 수학 내용 포함)
