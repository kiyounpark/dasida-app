# 진단 미니 카드 + 학습 노트 컬렉션 + 마일스톤

**날짜:** 2026-04-26
**상태:** 설계 완료
**관련 스펙:**
- `2026-04-25-unified-result-and-report-design.md` (현재 모의고사 → 진단 → 리포트 흐름)
- `2026-04-25-diagnosis-partial-exit-to-result-design.md` (부분 완료 시 리포트 자동 이동)

## 문제 정의

모의고사 풀고 약점 분석 흐름이 사용자에게 부담을 준다.

### 현재 흐름

```
모의고사 풀이
  → 채점 결과 (오답 타일 그리드)
  → 진단 세션 (오답 N개 전부 스와이프하며 분석)
  → 모든 오답 분석 완료 시 → 약점 종합 리포트
```

### 관찰된 약점

1. **시작 장벽** — 30문제 중 15개 틀리면 타일 15개를 보고 "다 해야 해?" 하고 뒤로감
2. **중간 공허감** — 5~6개 분석하다가 지쳐서 나가도 리포트는 안 나옴. 뭘 얻었는지 불명확
3. **즉각 보상 부재** — 진단 1개 완료 후 다음 문제로 자동 전환. 사용자가 "방금 뭘 얻었지?" 라는 인식 순간이 없음
4. **목표 거리감** — 진행 바 5/15만으로는 "언제 다 해" 느낌이 강함

데이터 기반 관찰은 아직 없음. 제품 직관 기반 가설.

### 거짓 보상의 위험

부분 단계에서 "1위 약점: 미분"을 보여주는 안(Progressive Unlock)도 검토했으나 기각. 이유:

- 5문제 분석한 1위 약점이 15문제 분석 후 3위로 떨어지면 학생이 학습 전략을 잘못 세움
- 빈도 숫자("5문제 중 3문제에서 발견")도 사용자가 머릿속에서 정렬해 동일 위험 재현
- 수능 준비 앱의 신뢰성 훼손은 작은 UX 실수가 아님

→ 부분 단계에 종합 정보를 절대 보여주지 않는다는 원칙으로 진행.

## 설계

### 핵심 아이디어

진단과 종합 리포트를 **다른 상품**으로 분리한다.

| 단계 | 산출물 | 정보 성격 |
|---|---|---|
| 진단 1개 완료 시점 | 미니 완료 카드 | 그 문제 한정 피드백 (개별, 사실) |
| 진행 중 (매 진단) | 학습 노트 컬렉션 (점 게이지) | 카운트만 (사실, 거짓 없음) |
| 1/3, 2/3 도달 | 마일스톤 셀레브레이션 | 새 정보 0, 감정 보상만 |
| 전체 완료 시점 | 약점 종합 리포트 | 1위/2위/3위 + 학습 추천 (완전 데이터) |

부분 단계에 **순위 비슷한 어떤 정보도 추론 가능하게 노출하지 않는다**.

### 화면별 변경

#### 1. 채점 결과 화면 (`features/quiz/exam/screens/exam-result-screen-view.tsx`)

- 카피 변경: "약점 분석 시작하기" CTA 위/아래에 "학습 노트를 모아보세요" 프레임 카피 한 줄
- 하단 microcopy: "15장 노트 모두 모으면 종합 리포트가 열려요"
- 거짓 약속 카피 금지: "3개만 해도 리포트 열려요" 같은 부분 보상 약속 절대 안 함
- 오답 타일 그리드는 그대로 유지

#### 2. 진단 세션 (`features/quiz/exam/screens/exam-diagnosis-session-screen.tsx` + `exam-diagnosis-screen.tsx`)

- 기존 흐름 유지: 방식 선택 → diagnosis tree (Choice/Explain/Check)
- **변경: 진단 1개 완료 시 즉시 다음 문제로 자동 이동하지 않음**. 미니 완료 카드를 한 박자 띄움
- 상단 진행 바와 "5/15" 표기는 유지

#### 3. ★ 미니 완료 카드 (NEW)

진단 1개가 끝나는 시점에 표시. 두 영역으로 구성:

```
┌─────────────────────────────────┐
│           ✓ 분석 완료            │
│            9번 문제              │
│                                 │
│  ┌───────────────────────────┐  │
│  │ 오답 패턴                  │  │
│  │ 계산 실수 — 부호 처리      │  │
│  │ 시간 압박 상황에서 부호를  │  │
│  │ 놓쳤어요. 검산 한 번이면   │  │
│  │ 잡을 수 있어요.            │  │
│  └───────────────────────────┘  │
└─────────────────────────────────┘

┌─────────────────────────────────┐
│ 📔 학습 노트       5장            │
│                                 │
│ ●●●●●○○○○○○○○○○                │
│                                 │
│ 10장 더 모으면 종합 리포트 ✦    │
└─────────────────────────────────┘

[잠시 쉬기]    [다음 문제 →]
```

- **상단 — 미니 완료 카드:** 그 문제 한정 피드백. "오답 패턴" + 한두 줄 설명. 이미 diagnosis tree 결과로 산출 가능한 정보 사용. 새 콘텐츠 작성 없음.
- **하단 — 학습 노트 컬렉션:** 15개 점 게이지. 완료된 만큼 채워짐. 카운트만 표시. 종류 분류 없음, 순위 없음.
- **두 버튼:** "잠시 쉬기" → 홈, "다음 문제 →" → 다음 진단

#### 4. 마일스톤 셀레브레이션 (5장, 10장 도달 시)

5장 또는 10장째 진단이 완료되는 그 미니 카드 자리에 한 번만 마일스톤 배너 형태로 변형:

```
┌─────────────────────────────────┐
│            🌱                    │
│         1/3 도달                 │
│   노트 5장 모았어요              │
│   여기까지 잘 왔어요              │
│           5 / 15                 │
└─────────────────────────────────┘

📔 노트 컬렉션 (5장 채워진 점)

[잠시 쉬기]    [계속하기 →]
```

- 새 정보 풀어주지 않음. 그저 시각적 축하 + 응원
- 1/3 (5장), 2/3 (10장) 두 번. 3/3은 종합 리포트 자체가 클라이맥스라 별도 마일스톤 불필요
- 한 번만 표시. 이후 진단에서는 일반 미니 카드로 복귀

#### 5. 홈 화면 — 모의고사 분석 진행 중 모드 (`features/quiz/components/quiz-hub-screen-view.tsx`)

진단 미완료 상태(`pendingDiagnosisResume` 존재)일 때 홈이 다른 모드로 전환된다.

**홈 분기 규칙 — 액션 있는 것만, 모순되는 건 숨김**

| 요소 | 평소 모드 | 모의고사 분석 진행 중 모드 |
|---|---|---|
| `BrandHeader` | 보임 | 보임 |
| `PosterTitleBanner ("학습 여정")` | 보임 | **숨김** |
| `JourneyBoard` | 보임 (메인) | **숨김** (이미 완료한 사용자) |
| `JourneyCtaButton` | 보임 | **숨김** |
| 모의고사 시작하기 카드 | 보임 | **숨김** (이미 진행 중, 액션 모순) |
| `ReviewHomeCard` (복습 있는 날) | 보임 | **유지 + 최상단으로 이동** |
| `NoReviewDayCard` (복습 없는 날) | 보임 | **숨김** ("쉬어도 돼요"는 액션 없음, 노이즈) |
| 분석 진행 중 카드 (NEW) | 없음 | **메인** |
| 모은 노트 리스트 (NEW) | 없음 | 보임 |
| `HomeWeaknessSection` (지난 분석 약점) | 보임 | 유지 (과거 데이터, 모순 없음) |

**복습 있는 날 — 홈 순서:**

```
[BrandHeader]
[ReviewHomeCard] ← 1번. 빠른 일상, 위
[모의고사 분석 진행 중 카드] ← 2번. 큰 일, 아래 (시각적으로 무겁게)
[모은 노트 리스트]
[HomeWeaknessSection]
```

**복습 없는 날 — 홈 순서:**

```
[BrandHeader]
[모의고사 분석 진행 중 카드] ← 1번. 자연스럽게 메인 자리
[모은 노트 리스트]
[HomeWeaknessSection]
```

**왜 복습이 위인가:**

1. **복습은 매일의 앵커 습관** — 모의고사 분석은 매일은 아님. 일상이 위, 이벤트가 아래
2. **작은 성취 먼저** — 5분이면 끝나는 복습이 위에 있으면 사용자가 모멘텀을 얻고 모의고사 분석에 들어감
3. **시간 민감도** — 에빙하우스 복습은 미루면 스케줄 깨짐. 모의고사 분석은 며칠 미뤄도 손실 없음

분석 진행 중 카드 자체 디자인:

```
┌─────────────────────────────────┐
│ 📔 모의고사 분석 진행 중         │
│ 2026 수능 수학                   │
│                                 │
│ ┌─────────────────────────┐    │
│ │ 📔 학습 노트     5 / 15  │    │
│ │ ●●●●●○○○○○○○○○○         │    │
│ └─────────────────────────┘    │
│                                 │
│ [이어서 분석하기 →]              │
└─────────────────────────────────┘
```

- 그라디언트 그린(`#C8EAC8 → #EDF7ED`)으로 시각적 무게. 복습 카드보다 큼. 위에 있지 않아도 안 잃어버림
- 노트 컬렉션 점 게이지 미니 버전. 진행 자산 시각화
- 모은 노트 목록은 탭 가능 (각 노트의 미니 카드 다시 보기). 푼 만큼은 손해 없음

#### 6. 종합 리포트 (`features/quiz/components/quiz-result-report-view.tsx`)

15/15 완주 시점에 처음 등장. 기존 리포트 구조 유지하되 시작 부분에 클라이맥스 배너 추가:

```
🎉 15장 노트 모두 수집!
전체 분석 완료 · 약점 3개 발견

●●●●●●●●●●●●●●● (15개 모두 채워진 점 게이지)

🔴 1위 약점 — 미분 / 도함수 활용
🟠 2위 약점 — 수열의 합 · 점화식
🟢 3위 약점 — 확률과 통계

[약점 기반 연습문제 →]
```

- 1위/2위/3위 라벨은 **이 화면에서 처음** 등장
- 완전 데이터 기반이라 순위 흔들리지 않음

## 데이터 모델

새 데이터 추가 없음. 기존 구조로 충분:

- 진단 결과: `diagnosedProblems` (`{problemNumber: weaknessId}`) — `features/quiz/exam/exam-diagnosis-progress.ts`
- 미니 카드 콘텐츠: 각 진단의 마지막 단계(Check 노드) 응답을 그대로 사용
- 노트 컬렉션 카운트: `Object.keys(diagnosedProblems).length` 또는 진단 큐 진행 상태에서 도출
- 노트 목록 (홈): `diagnosedProblems` 키 순회 → 각 문제의 weakness label

기존 `compute-exam-top-weaknesses.ts`는 종합 리포트 단계에서만 호출. 부분 단계에서는 호출 안 함.

## 엣지 케이스

| 케이스 | 처리 |
|---|---|
| 오답 1~2개 (마일스톤 도달 안 됨) | 마일스톤 배너 안 뜸. 일반 미니 카드만. 1~2개여도 종합 리포트는 정상 동작 |
| 오답 5개 정확히 (마일스톤=완주) | 5장 도달 시 마일스톤 배너 후 다음 문제 없으면 곧바로 종합 리포트로 |
| 진단 도중 앱 종료 후 재진입 | 기존 `pendingDiagnosisResume` 동작 그대로. 노트 컬렉션 카운트는 `diagnosedProblems`에서 복원 |
| 미니 카드 보고 "잠시 쉬기" 누름 | `pendingDiagnosisResume` 저장 후 홈 이동. 다음에 "이어서 분석하기"로 복귀 |
| 같은 문제 재진단 | 기존 정책 따름 (한 번 진단된 문제는 재진단 안 함). 노트는 1장으로 카운트 |
| diagnosis tree에서 Check 노드 응답이 없는 경우 | 미니 카드의 "오답 패턴" 영역은 fallback 표시. "분석 완료" 한 줄 + 진행 표시만 |

## 디자인 원칙 재확인

1. **부분 단계에서 종합 정보 노출 금지** — 빈도, 분류, 순위, 추정값 모두 안 보여줌
2. **카운트는 사실** — 노트 수만 보여줌. 순위 추론 불가
3. **마일스톤은 정보 0** — 새로운 콘텐츠 풀어주지 않음. 감정 보상만
4. **개별 진단의 가치를 명확히** — 미니 카드로 그 문제 한정 피드백을 한 번 인식시킴
5. **나가도 손해 없음** — 모은 노트는 홈에서 다시 볼 수 있음

## 열린 질문

1. **마일스톤 위치 5/10 vs 33%/67%** — 오답 수가 적을 때(예: 6개) 5장 마일스톤이 사실상 완주 직전이라 어색할 수 있음. 절대값(5, 10) 대신 비율(33%, 67%)로 갈지? → 구현 단계에서 한 번 더 결정.
2. **미니 카드 "오답 패턴" 텍스트 소스** — diagnosis tree Check 노드의 응답을 그대로 쓸지, 별도 요약 콘텐츠 데이터를 만들지. 기존 데이터 재사용이 우선이나 표현이 어색하면 별도 매핑 검토.
3. **마일스톤 배너 노출 횟수** — 한 번만? 매번 5의 배수마다? → 한 번만으로 시작. 사용자 피로도 우려.

이 세 항목은 구현 계획 단계에서 확정한다.
