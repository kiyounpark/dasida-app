# Design: 홈화면(QuizHub) iPad 레이아웃 100% 채움 개선

## 한 줄 요약

아이패드 홈화면(QuizHub)에서 콘텐츠가 화면의 ~60-70%만 차지하고 좌우 여백이 크게 남는 문제를, 컨테이너 maxWidth를 스크린 폭의 92%로 동적 계산하고 개별 컴포넌트 상한값을 상향하여 해결한다.

## 문제 정의

`tabletPosterScreen.maxWidth: 720`이 정적 고정값이라 iPad Pro 13" (1024pt 세로, 1366pt 가로)에서 콘텐츠가 화면의 ~70%만 점유한다. 그 안에 `width: '100%'` 카드들도 컨테이너가 좁으면 따라서 좁아진다. `NoReviewDayCard`는 자체 `maxWidth: 460`이 있어 더 심하다.

| 기기 | 화면 폭 | 현재 컨테이너 | 점유율 |
|------|---------|--------------|--------|
| iPad mini 6 | 744pt | 720pt | 96.8% ✅ |
| iPad Pro 11" | 834pt | 720pt | 86.4% △ |
| iPad Pro 13" | 1024pt | 720pt | 70.3% ❌ |
| iPad Pro 13" 가로 | 1366pt | 720pt | 52.7% ❌ |

## 목표

- iPad mini~Pro 13" 세로/가로 모두에서 콘텐츠가 화면의 88-94% 점유
- iPhone 레이아웃 회귀 없음
- 1열 흐름(복습카드 → 여정판 → 약점 → CTA) 유지

## 변경 범위

### 1. `features/quiz/components/quiz-hub-screen-view.tsx`

**핵심 변경**: `tabletPosterScreen.maxWidth` 정적 720 → 동적 계산값

```tsx
// 추가
import { useWindowDimensions } from 'react-native';

// QuizHubScreenView 함수 내부
const { width: screenWidth } = useWindowDimensions();
const tabletContainerMaxWidth = isTablet
  ? Math.min(screenWidth * 0.92, 1040)
  : undefined;
```

`tabletPosterScreen` 스타일에서 `maxWidth: 720` 제거하고,  
ScrollView `contentContainerStyle`에서 inline으로 적용:

```tsx
isTablet && { maxWidth: tabletContainerMaxWidth, width: '100%', alignSelf: 'center' }
```

**CTA 버튼**: `isTablet && { maxWidth: 480 }` → `isTablet && { maxWidth: 600 }`

### 2. `features/quiz/components/poster-title-banner.tsx`

```tsx
// 변경 전
heroFrameWrapTablet: { maxWidth: 560 }

// 변경 후
heroFrameWrapTablet: { maxWidth: 700 }
```

히어로 배너는 종횡비 고정(`HERO_FRAME_ASPECT_RATIO = 1542/437`)이라 높이가 자동 따라오므로 동적 비율 대신 상한만 올린다.

### 3. `features/quiz/components/no-review-day-card.tsx`

현재 `maxWidth: 460` 고정. `useIsTablet` 추가 후 태블릿에서 maxWidth를 제거해 컨테이너 폭에 맞게 stretch.

```tsx
import { useIsTablet } from '@/hooks/use-is-tablet';

// 컴포넌트 내부
const isTablet = useIsTablet();

// 카드 wrapper style
[styles.card, isTablet && styles.cardTablet]

// StyleSheet.create에서는 maxWidth 속성 자체를 제거하고,
// 컴포넌트에서 조건부 인라인 스타일로 처리:
// [styles.card, isTablet && { maxWidth: undefined }]
// React Native는 undefined 스타일 속성을 무시하므로 maxWidth 제약이 해제됨
```

### 4. `features/quiz/components/journey-board.tsx`

현재 태블릿 상한: `Math.min(screenWidth * 0.7, 640)`. 여정판은 VIEWBOX 종횡비(768×H)가 고정이라 너무 넓으면 세로로 압도. 상한만 소폭 상향:

```tsx
// 변경 전
const boardMaxWidth = isTablet
  ? Math.min(screenWidth * 0.7, 640)
  : isCompactLayout ? 430 : 470;

// 변경 후
const boardMaxWidth = isTablet
  ? Math.min(screenWidth * 0.7, 680)
  : isCompactLayout ? 430 : 470;
```

## 변경하지 않는 것

| 항목 | 이유 |
|------|------|
| ReviewHomeCard | `width: '100%'`라서 컨테이너 확장 시 자동 해결 |
| HomeWeaknessSection | 마찬가지 |
| ExamAnalysisResumeCarousel | 이미 `containerWidth` 기반 동적 처리 |
| iPhone 레이아웃 | `isTablet` 분기 안에서만 변경 |
| 1열 구조, 패딩, 색상 등 | 변경 없음 |
