# Design: 퀴즈 세션 iOS 스와이프 뒤로가기 차단

Generated by /brainstorming + /office-hours on 2026-04-23
Branch: main
Status: DRAFT

## Problem Statement

iOS에서 왼쪽→오른쪽 스와이프 제스처로 퀴즈 세션 도중 의도치 않게 화면을 나갈 수 있다.
사용자가 원하는 것: 스와이프로는 절대 나갈 수 없고, 헤더 왼쪽 아이콘 버튼을 통해서만 나가기 확인 모달이 뜨도록.

## 적용 대상 화면

| 화면 | 라우트 | 단계 |
|---|---|---|
| 10문제 풀기 (퀴즈 풀기 단계) | `/quiz/diagnostic` | `DiagnosticQuizStage` |
| 10문제 풀기 (오답 진단 단계) | `/quiz/diagnostic` | `isDiagnosing === true` |
| 약점 연습 | `/quiz/practice` | 모든 모드 (weakness / challenge / review) |

## 현재 상태 (변경 전)

- `QuizSolveHeader` 좌상단에 `← Back` 버튼 존재 → 모달 열기 (`onBackPress`) ✅ 이미 구현됨
- `QuizSolveExitConfirmModal` 렌더링 ✅ 이미 구현됨
- `DiagnosisDarkHeader`의 `← 뒤로` 버튼 → 진단 종료 모달 ✅ 이미 구현됨
- **단, `gestureEnabled: false` 없음 → 스와이프로 모달 없이 바로 나가짐** ❌

## 설계

### 변경 1: 스와이프 차단 (`app/quiz/_layout.tsx`)

```tsx
<Stack.Screen name="diagnostic" options={{ gestureEnabled: false }} />
<Stack.Screen name="practice"   options={{ gestureEnabled: false }} />
```

### 변경 2: 헤더 "Back" 텍스트 제거 (`features/quiz/components/quiz-solve-header.tsx`)

```tsx
// Before
<IconSymbol ... />
<Text style={styles.backLabel}>Back</Text>

// After
<IconSymbol ... />
{/* 텍스트 제거 — 아이콘만으로 충분 */}
```

`backLabel`, `backLabelCompact` 스타일도 함께 삭제.

## 나가기 플로우 (변경 후)

```
[스와이프 시도]    → gestureEnabled: false → 아무 일 없음
[헤더 아이콘 탭]  → onBackPress / onBack → 확인 모달 표시
  ├── "계속 풀기" / "계속하기" → 모달 닫기, 세션 재개
  └── "나가기"                 → resetSession() or finishDiagnosis()
                                 → router.replace('/(tabs)/quiz')
```

## 변경 범위

| 파일 | 변경 내용 | 크기 |
|---|---|---|
| `app/quiz/_layout.tsx` | `gestureEnabled: false` 2줄 추가 | 극소 |
| `features/quiz/components/quiz-solve-header.tsx` | `<Text>Back</Text>` 삭제, 관련 스타일 삭제 | 극소 |

**건드리지 않는 것**: 훅 로직, 모달 컴포넌트, 세션 상태, 라우팅 — 이미 완성된 코드.

## 검증 항목

- [ ] iOS 시뮬레이터: 스와이프해도 화면 안 나가짐 (diagnostic, practice)
- [ ] 헤더 아이콘 탭 → 확인 모달 표시
- [ ] 모달 "계속 풀기" → 세션 재개 (상태 유지)
- [ ] 모달 "나가기" → 퀴즈 허브로 이동, 세션 리셋
- [ ] 진단 단계(isDiagnosing)에서도 동일 동작
- [ ] 헤더 아이콘이 텍스트 없이 올바르게 렌더링됨
- [ ] 다른 화면(result, feedback, step-complete)에는 영향 없음

## 미적용 화면 (의도적 제외)

- `challenge`, `review` 모드: 약점 연습과 같은 `practice` 라우트이므로 자동 포함됨
- `exam/solve`: 이번 스코프 제외 (별도 작업)
- `result`, `feedback`, `step-complete`: 뒤로가기 허용 유지

## Premises

1. 스와이프 차단은 iOS만 해당 (`gestureEnabled`는 iOS 제스처 전용)
2. Android 물리 뒤로가기는 이번 스코프 제외
3. 기존 나가기 모달/버튼 로직은 이미 올바르게 구현됨 — 재작성 불필요
