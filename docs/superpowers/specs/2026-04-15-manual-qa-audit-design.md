# Manual QA Audit — 설계 문서

**날짜**: 2026-04-15
**목적**: 실기기 수동 탐색을 통해 버그와 기능 공백을 체계적으로 발견하고 분류하는 워크플로우 설계

---

## 배경

앱이 어느 정도 기능을 갖춘 시점에서, 실기기를 직접 탐색하며 놓친 버그와 없는 기능을 발견하고자 한다. 탐색 중 뭘 봐야 할지 기준이 없거나, 발견한 내용을 어떻게 정리할지 몰라 탐색이 비효율적이 되는 문제를 해결한다.

---

## 탐색 시나리오

새 유저 시점으로 3개 시나리오를 순서대로 탐색한다.

### 시나리오 A — 첫 진입 & 단원 진단
1. 앱 첫 실행 (캐시 클리어 또는 로그아웃 상태)
2. 온보딩 화면
3. 로그인
4. 퀴즈 탭 진입
5. 단원 진단 시작 → 문제 풀기
6. 진단 완료 → 결과 확인
7. 복습 세션 진입
8. AI 피드백 주고받기

### 시나리오 B — 모의고사 흐름
1. 모의고사 탭 진입
2. 문제 풀기
3. 결과 화면 확인
4. 오답 타일 그리드 확인
5. 오답 진단 진행

### 시나리오 C — 주변 화면 탐색
1. 히스토리 탭 — 기록 목록 확인
2. 프로필 탭 — 학년 설정
3. 앱 정보 — 개인정보처리방침 링크 동작
4. FoundingMemberCard 표시 여부 확인

---

## 체크리스트 관점

각 시나리오의 모든 단계에서 아래 5가지 관점으로 확인한다.

| 관점 | 확인 내용 |
|------|----------|
| **흐름** | 버튼을 눌렀을 때 다음 화면으로 잘 이동하는가 |
| **데이터** | 내용이 제대로 표시되는가 (빈 화면, 로딩 무한, 엉뚱한 값 없는지) |
| **UI** | 텍스트 잘림, 레이아웃 깨짐, 버튼 겹침 없는가 |
| **빈 상태** | 기록이 없을 때, 처음 사용할 때 화면이 자연스러운가 |
| **없는 기능** | "이게 있어야 자연스러운데 없네" 싶은 것 |

---

## 메모 형식

탐색 중 발견 즉시 아래 형식으로 메모한다.

```
[시나리오] [화면이름] [관점]
내용: 어떤 문제인지 한 줄
```

예시:
```
[A] 복습세션 [흐름]
내용: AI 응답 후 "다음 단계" 버튼이 안 보임

[B] 모의고사 결과 [없는 기능]
내용: 오답만 모아서 다시 풀기 기능 없음
```

## 결과 분류 및 저장

탐색 후 메모를 이 대화창에 붙여넣으면 자동 분류한다.

### 버그 → `docs/QA-findings.md`

```markdown
## 버그 수정 큐

| # | 화면 | 내용 | 심각도 | 상태 |
|---|------|------|--------|------|
| 1 | 복습세션 | AI 응답 후 다음 버튼 안 보임 | 높음 | 미수정 |
```

심각도 기준:
- **높음**: 앱을 정상적으로 사용하지 못함
- **중간**: 불편하지만 우회 가능
- **낮음**: 사소한 시각적 문제

### 기능 공백 → Notion "DASIDA 개발 기록"

각 공백 항목을 새 페이지로 추가한다.
- 상태: `기획중`
- 제목: 기능명 (예: "오답 모아서 다시 풀기")
- 본문: 어느 시나리오에서 발견했는지 + 왜 필요한지 한 줄

## 전체 워크플로우

```
실기기 탐색 (시나리오 A → B → C)
         ↓
핸드폰 메모앱 또는 Notion 모바일로 메모
         ↓
Claude 대화창에 붙여넣기
         ↓
버그 → docs/QA-findings.md 추가
기능 공백 → Notion 페이지 생성
```
