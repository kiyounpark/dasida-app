## 한 줄 요약

진단 완료 시 Firestore에 `diagnosis_completed` 이벤트를 기록한다.
7일 이내 재진단 여부로 리텐션을 계산한다.

---

## 배경

유료화 전략 spec(2026-04-14-monetization-strategy.md)에서 정한 핵심 지표:

> "진단을 한 번 완료한 학생 중 7일 안에 다시 진단을 한 비율"

이 데이터가 없으면 유료화 시점을 판단할 수 없다.
Firebase Analytics 웹 SDK는 React Native에서 세션 추적이 불안정하고,
`@react-native-firebase`는 현재 웹 SDK 기반 Firebase와 별도 SDK라 출시 전 설치 비용이 크다.
→ 기존에 세팅된 Firestore를 사용한 커스텀 이벤트 로그로 구현한다.

---

## Firestore 데이터 구조

```
users/{firebaseUid}/events/{autoId}
  eventName:     "diagnosis_completed"
  source:        "exam" | "unit"
  weaknessId:    string
  examId:        string | null   (exam 전용, unit이면 null)
  problemNumber: number | null   (exam 전용, unit이면 null)
  completedAt:   Timestamp       (serverTimestamp)
```

**경로 규칙:** `firebaseUid`는 `profile.accountKey`에서 `"user:"` 접두사를 제거한 값.
기존 `firestore-learner-profile-store.ts`와 동일한 `users/{uid}/` 패턴.

## 새 파일

### `features/analytics/diagnosis-analytics.ts`

- `logDiagnosisCompleted(params)` 함수 하나만 export
- **fire-and-forget**: UI를 블록하지 않음, `.catch(() => {})` 로 에러 무시
- Firestore 인스턴스는 기존 Firebase app에서 `getFirestore(getApp())`으로 가져옴

```ts
import { addDoc, collection, getFirestore, serverTimestamp } from 'firebase/firestore';
import { getApp } from 'firebase/app';

export type DiagnosisCompletedSource = 'exam' | 'unit';

interface LogDiagnosisCompletedParams {
  accountKey: string;           // "user:{firebaseUid}" 형태
  source: DiagnosisCompletedSource;
  weaknessId: string;
  examId?: string;
  problemNumber?: number;
}

export function logDiagnosisCompleted(params: LogDiagnosisCompletedParams): void {
  const uid = params.accountKey.startsWith('user:')
    ? params.accountKey.slice(5)
    : params.accountKey;

  const db = getFirestore(getApp());

  addDoc(collection(db, 'users', uid, 'events'), {
    eventName: 'diagnosis_completed',
    source: params.source,
    weaknessId: params.weaknessId,
    examId: params.examId ?? null,
    problemNumber: params.problemNumber ?? null,
    completedAt: serverTimestamp(),
  }).catch(() => {
    // 리텐션 로깅 실패는 UX에 영향 없이 무시
  });
}
```

## 호출 위치

### 1. 모의고사 진단 (exam)

**파일:** `features/quiz/exam/hooks/use-exam-diagnosis.ts`
**위치:** `useEffect` 내부, `setIsDone(true)` 직후 (line ~277)
**사용 가능한 값:** `profile.accountKey`, `weaknessId`, `draft.methodId` 기반 flow, `examId`, `problemNumber`

```ts
setIsDone(true);
logDiagnosisCompleted({
  accountKey: profile.accountKey,
  source: 'exam',
  weaknessId,
  examId,
  problemNumber,
});
```

### 2. 단원 진단 (unit)

**파일:** `features/quiz/hooks/use-diagnostic-screen.ts`
**위치:** `onFinalConfirm` 함수 내 `status: 'completed'` 설정 직후
**필요 추가:** `useCurrentLearner()` 훅 호출 (현재 `profile` 없음)

```ts
const { profile } = useCurrentLearner();
// ...
// onFinalConfirm 내부:
if (profile) {
  logDiagnosisCompleted({
    accountKey: profile.accountKey,
    source: 'unit',
    weaknessId: activeNode.weaknessId,
  });
}
```

`profile`이 null이면 (미인증 게스트) 조용히 건너뜁니다. 로그가 없는 것이지 에러가 아닙니다.

## 리텐션 계산 방법

Firebase 콘솔 → Firestore → 컬렉션 그룹 쿼리 (`events`):

1. `eventName == "diagnosis_completed"` 인 문서 전체 조회
2. 유저별로 첫 번째 `completedAt` 날짜 기록
3. 7일 이내에 두 번째 `completedAt` 있으면 → 리텐션 ✅

**판정 기준 (유료화 전략 spec 기준):**

| 7일 리텐션 | 다음 행동 |
|-----------|----------|
| 30% 이상 | 반복 유저 3명 인터뷰 → 유료화 설계 시작 |
| 10~30% | 이탈 지점 파악 |
| 10% 미만 | 제품 개선 먼저 |

## 제외 범위

- `app_open` 이벤트 추적 없음 (진단 재사용이 핵심 지표)
- 리텐션 대시보드 UI 없음 (Firebase 콘솔에서 수동 확인)
- `@react-native-firebase` 마이그레이션 없음 (유저 100명 이후 재검토)

## Success Criteria

- 진단 완료 후 Firestore `users/{uid}/events`에 문서 생성 확인
- 로깅 실패 시 앱 크래시 없음 (fire-and-forget)
- 7일 이내 재진단 시 동일 `uid`로 두 번째 문서 생성 확인
