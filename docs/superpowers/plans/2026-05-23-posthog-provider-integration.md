## 배경

- 현재 `features/analytics/posthog.ts`는 PostHog SDK를 싱글톤으로 직접 호출 (`PostHogProvider` 없음).
- 이벤트 캡쳐는 잘 되지만 **Survey, Feature Flags의 자동 UI 렌더링은 Provider 없으면 안 됨.**
- spec에서 결정된 인터뷰 모집 채널 A를 작동시키려면 Provider 통합 필수.

## 목표

1. `PostHogProvider`로 다시다 앱 전체를 감싼다.
2. 기존 싱글톤 호출 코드(`capture`, `identify`, `reset`)와 충돌 없이 공존.
3. PostHog 대시보드의 Survey가 트리거 조건 만족 시 앱 안에서 자동 popover로 뜬다.
4. 응답률 보강을 위해 Survey 카피에 "기프티콘 추첨" 메리트 + 마지막 질문에 연락처 수집 추가.

## 작업 시간 추산

## 단계별 작업

### 1. 의존성 확인

- `posthog-react-native` ^4.45.5 이미 설치됨 (package.json). 추가 설치 없음.
- ⚠️ `PostHogProvider`는 같은 패키지에서 import 가능. 네이티브 의존성 추가 없음 → **prebuild 불필요.**

### 2. PostHogProvider 통합 (`app/_layout.tsx`)

```tsx
import { PostHogProvider } from 'posthog-react-native';

const POSTHOG_API_KEY = process.env.EXPO_PUBLIC_POSTHOG_API_KEY;
const POSTHOG_HOST = process.env.EXPO_PUBLIC_POSTHOG_HOST ?? 'https://us.i.posthog.com';

// 기존 RootLayout return 가장 바깥쪽에 추가
return (
  <PostHogProvider
    apiKey={POSTHOG_API_KEY}
    options={{
      host: POSTHOG_HOST,
      sessionExpirationTimeSeconds: 30 * 60,
      flushInterval: 5000,
      flushAt: 20,
      captureAppLifecycleEvents: false,
      enableSessionReplay: false,
    }}
    autocapture={false}
  >
    {/* 기존 RootLayout 내용 */}
  </PostHogProvider>
);
```

**주의 사항:**
- `autocapture={false}` 필수 — GA4와 의미 중복 방지.
- options는 `posthog.ts` 싱글톤과 **완전히 동일하게** 맞춰야 함 (세션, flush, lifecycle).
- API key 없으면 Provider는 no-op 동작 (env 누락 환경 안전).

### 3. 싱글톤과 Provider의 공존 처리

**옵션 A (권장): 기존 싱글톤 유지, Provider는 Survey/Flags용으로만**
- `features/analytics/posthog.ts`의 `getPostHog()`/`capture()` 그대로 둠.
- Provider는 Survey 렌더링 + (선택) Feature Flag만 담당.
- 트레이드오프: 같은 API key로 두 인스턴스가 뜰 수 있음. PostHog SDK가 내부적으로 dedup 처리하는지 확인 필요.

**옵션 B: 싱글톤 제거, `usePostHog()` 훅으로 전환**
- 더 깔끔하지만 `log-event.ts` 등 모듈 레벨 호출 코드 다 리팩터.
- 작업량 2~3시간 추가.

→ **이번 작업은 옵션 A로 진행.** Survey 작동 검증 후 필요시 B로 마이그레이션.

### 4. Survey 카피 / 질문 수정 (PostHog 대시보드)

"다시다 인터뷰 모집 (draft)" Survey를 열고:

- **타이틀**: "다시다 5분 인터뷰, 함께 해주실래요?"
- **본문**:
  > 다시다를 어떻게 쓰고 계신지 5분 정도 얘기 들려주시면,
  > 추첨을 통해 **스타벅스 기프티콘**을 보내드려요 🎁
- **마지막 질문 추가**: "당첨 시 연락드릴 카톡 ID 또는 전화번호를 적어주세요" (open text, optional)
- **Display conditions**: spec에 명시된 트리거 그대로
  - `diagnosis_completed ≥ 2` AND `mock_exam_completed ≥ 1` AND `review_completed ≥ 1`
  - 빈도: 한 번만 (once per user)
- **표시 방식**: 하단 popover (덜 침습적)

### 5. 기프티콘 추첨 정책 (박기윤이 결정 후 plan에 채워넣기)

- [ ] 추첨 빈도: ___ (예: 매월 1회 1명, 분기 1회 3명 등)

- 구현: `react-native-screen-capture` 또는 Share API 핸들러에서 `capture('share_or_screenshot', { source: ... })` 호출
- 우선순위: P1 (Survey 통합 후 별도 작업)
